🔑 eSIMACCESS API ENTEGRASYONU (DOKÜMAN: docs.esimaccess.com)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📡 ORTAK HEADERS (her istekte zorunlu):
RT-AccessCode: process.env.ESIM_ACCESS_CODE
Content-Type: application/json
Not: Signature/RequestID/Timestamp gerekiyorsa crypto ile dinamik üret.

📦 1) Paket Listesi:
POST https://api.esimaccess.com/api/v1/open/package/list
Body: { "locationCode": "", "type": "BASE" }
• locationCode boş = tüm paketler
• Fiyat dönüşü: price / 10000 = USD
• Günde 2 kez Vercel Cron ile çek ve Firestore'a yaz

🛒 2) Sipariş Oluşturma:
POST https://api.esimaccess.com/api/v1/open/esim/order
Body: { "transactionId": "crypto.randomUUID()", "amount": 15000, "packageInfoList": [{ "packageCode": "AU_1_7", "count": 1, "price": 15000 }] }
• Başarılı yanıtta orderNo döner → Firestore orders/{uid}/{orderNo} olarak sakla

📲 3) QR & Detay Sorgulama:
POST https://api.esimaccess.com/api/v1/open/esim/query
Body: { "orderNo": "...", "pager": { "pageNum": 1, "pageSize": 50 } }
• Yanıt: qrCodeUrl, shortUrl, ac, iccid, esimStatus, activateTime, expiredTime
• qrCodeUrl direkt <img src=""> olarak gösterilebilir

💰 4) Bakiye Sorgulama (Admin panelinde göstereceğim):
POST https://api.esimaccess.com/api/v1/open/balance/query
• Body boş, yanıt: balance / 10000 = USD

🔄 5) Top-Up:
POST https://api.esimaccess.com/api/v1/open/esim/topup
Body: { "iccid": "...", "packageCode": "...", "transactionId": "...", "amount": ... }

⏸️ 6) Yönetim:
POST /esim/suspend | /esim/unsuspend | /esim/revoke
Body: { "iccid": "..." }

⚠️ KURALLAR:
• transactionId her siparişte crypto.randomUUID() ile benzersiz üret
• Fiyatlar: eSIMAccess price / 10000 = USD. Ters işlem için * 10000
• API hatalarında otomatik iade: kullanıcı wallet'ine sipariş tutarı geri yatır
• Tüm API çağrıları /api/ içinde serverless fonksiyon olarak çalışacak

🚀 VERCEL DEPLOYMENT TALİMATLARI (README.md'ye EKLE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Replit'te proje oluştur. Tüm dosyaları yukarıdaki yapıda üret.
2. .env.example'den .env dosyası oluştur. Aşağıdaki değişkenleri doldur:
   FIREBASE_SERVICE_ACCOUNT_JSON=<service-account-json>
   ESIM_ACCESS_CODE=<code>
   ESIM_ACCESS_SECRET=<secret>
   PADDLE_VENDOR_ID=<vendor>
   PADDLE_SECRET_KEY=<key>
   FIREBASE_PROJECT_ID=<project>
3. vercel.json içindeki cron ve routing yapılandırmasını kontrol et.
4. Vercel CLI kur: npm i -g vercel
5. Deploy: vercel login → vercel --prod
6. Vercel Dashboard > Settings > Environment Variables'tan tüm .env değerlerini ekle.
7. Firestore Security Rules ve Storage Rules'ı Firebase Console'a yükle.
8. Paddle Webhook URL'sini https://<domain>.vercel.app/api/webhooks/paddle olarak ayarla.
9. Vercel Cron aktif olacak. Paket senkronizasyonu otomatik çalışacak.
